# v1.1 Governance Verification Procedure

## Purpose

This procedure defines reproducible governance verification for protected branch policy controls.

## Prerequisites

- Access to repository settings for verification.
- Access to CLI tooling used for governance checks.
- Access to capture UI screenshots for corroborating evidence.

## Branch Targets

- `master`
- `release/**`

## Canonical Verification Policy

- CLI command output is canonical when CLI and UI differ.
- UI screenshots are corroborating evidence and must be linked to CLI outputs.
